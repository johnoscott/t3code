# How It Works

We want to understand how this codebase wraps Claude Code to produce a GUI experience instead of a terminal-based one.

Specifically, we want to trace:

- How the GUI frontend communicates with the underlying Claude Code process.
- The layer(s) that translate between Claude Code's terminal/CLI surface and the GUI (protocols, IPC, streaming, etc.).
- Which Claude Code capabilities (tool calls, hooks, slash commands, MCP servers, permissions, sessions) are exposed through the GUI, and how.
- How interactive features normally handled in the terminal (prompts, approvals, tool permission requests, status updates) are rendered and controlled in the GUI.
- The lifecycle of a session: how it is started, persisted, resumed, and terminated from the GUI.
- Where the boundaries sit between the GUI shell, the wrapper/adapter layer, and Claude Code itself.

The output of this investigation should be a clear mental model of the architecture — enough to reason about where to add features, debug issues, or extend the GUI experience.
